#最佳实践 #redis #面试

```ad-summary
title: 总结

- 延迟双删：删缓存 → 更新 DB → 延迟 100-500ms 再删，延迟时间要大于一次读操作耗时
- 第二次删失败可以用重试或 MQ 异步删
- 强一致性用分布式锁：加锁 → 删缓存 → 更新 DB → 写新缓存 → 解锁
- 不想改业务代码可以用 Canal 监听 binlog 自动删缓存
```

本文是对 [[66.归档发布/03.缓存/缓存一致性\|缓存一致性]] 的详细补充，重点讲代码实现和方案选择。

## 1. 延迟双删怎么实现？
```java
public void updateUser(User user) {
    String cacheKey = "user:" + user.getId();
    
    // 1. 删除缓存
    redisTemplate.delete(cacheKey);
    
    // 2. 更新数据库
    userMapper.update(user);
    
    // 3. 延迟再删一次
    CompletableFuture.runAsync(() -> {
        try {
            Thread.sleep(500);
            redisTemplate.delete(cacheKey);
        } catch (InterruptedException e) {
            log.error("延迟双删失败", e);
        }
    });
}
```

### 1.1 延迟时间怎么定？

延迟时间要大于一次读操作的耗时：
```
延迟时间 = 读数据库时间 + 写缓存时间 + 网络延迟 + 余量

示例：
- 读数据库：50ms
- 写缓存：10ms
- 网络延迟：20ms
- 余量：20ms
总计：100ms，建议设 100-500ms
```

### 1.2 第二次删除失败怎么办？

**方案1**：重试
```java
@Retryable(maxAttempts = 3, backoff = @Backoff(delay = 1000))
public void deleteCache(String key) {
    redisTemplate.delete(key);
}
```

**方案2**：消息队列异步删除
```java
public void updateUser(User user) {
    String cacheKey = "user:" + user.getId();
    
    redisTemplate.delete(cacheKey);
    userMapper.update(user);
    
    // 发消息，让消费者延迟删缓存
    CacheDeleteMessage message = new CacheDeleteMessage(cacheKey, 500);
    mqProducer.send(message);
}

@RabbitListener(queues = "cache.delete.queue")
public void handleCacheDelete(CacheDeleteMessage message) {
    try {
        Thread.sleep(message.getDelayTime());
        redisTemplate.delete(message.getCacheKey());
    } catch (InterruptedException e) {
        log.error("异步删除缓存失败", e);
    }
}
```

## 2. 分布式锁怎么实现？

### 2.1 读操作加锁（防缓存击穿）
根据 [[66.归档发布/02.编码相关/Redisson最佳实践\|Redisson最佳实践]]，编写的代码。
```java
public User getUser(Long userId) {
    String cacheKey = "user:" + userId;
    String lockKey = "lock:user:" + userId;
    
    // 先查缓存
    User user = redisTemplate.opsForValue().get(cacheKey);
    if (user != null) {
        return user;
    }
    
    // 缓存没有，加锁
    RLock lock = null;
    try {
        lock = redissonClient.getLock(lockKey);
        // 等待 3s，不传 leaseTime 启用 Watchdog 自动续期
        boolean getLock = lock.tryLock(3, TimeUnit.SECONDS);
        if (!getLock) {
            throw new BizException("系统繁忙，请稍后重试");
        }
        
        // 双重检查，避免重复查 DB
        user = redisTemplate.opsForValue().get(cacheKey);
        if (user != null) {
            return user;
        }
        
        user = userMapper.selectById(userId);
        if (user != null) {
            redisTemplate.opsForValue().set(cacheKey, user, 30, TimeUnit.MINUTES);
        }
        return user;
    } catch (InterruptedException e) {
        Thread.currentThread().interrupt();
        throw new BizException("操作中断");
    } finally {
        // 只释放当前线程持有的锁
        if (lock != null && lock.isHeldByCurrentThread()) {
            lock.unlock();
        }
    }
}
```

### 2.2 写操作加锁（强一致性）

```java
public void updateBalance(Long userId, BigDecimal amount) {
    String cacheKey = "balance:" + userId;
    String lockKey = "lock:balance:" + userId;
    
    RLock lock = null;
    try {
        lock = redissonClient.getLock(lockKey);
        boolean getLock = lock.tryLock(3, TimeUnit.SECONDS);
        if (!getLock) {
            throw new BizException("系统繁忙，请稍后重试");
        }
        
        redisTemplate.delete(cacheKey);
        accountMapper.updateBalance(userId, amount);
        
        // 写入最新数据
        Account account = accountMapper.selectById(userId);
        redisTemplate.opsForValue().set(cacheKey, account, 30, TimeUnit.MINUTES);
    } catch (InterruptedException e) {
        Thread.currentThread().interrupt();
        throw new BizException("操作中断");
    } finally {
        if (lock != null && lock.isHeldByCurrentThread()) {
            lock.unlock();
        }
    }
}
```

## 3. 还有什么方案？

### 3.1 订阅 MySQL Binlog（Canal）

不改业务代码，监听数据库变更自动删缓存：

```java
@CanalEventListener
public class CacheInvalidateListener {
    
    @ListenPoint(schema = "mydb", table = "user")
    public void onUserChange(CanalEntry.EventType eventType, CanalEntry.RowData rowData) {
        if (eventType == CanalEntry.EventType.UPDATE || 
            eventType == CanalEntry.EventType.DELETE) {
            
            String userId = rowData.getAfterColumnsList().stream()
                .filter(col -> "id".equals(col.getName()))
                .findFirst()
                .map(CanalEntry.Column::getValue)
                .orElse(null);
            
            if (userId != null) {
                redisTemplate.delete("user:" + userId);
            }
        }
    }
}
```

### 3.2 设置合理的过期时间

过期时间是最后的兜底，就算删缓存失败，数据也不会一直脏着：

```java
// 不同数据设不同的过期时间
redisTemplate.opsForValue().set("user:" + userId, user, 30, TimeUnit.MINUTES);
redisTemplate.opsForValue().set("product:" + productId, product, 1, TimeUnit.HOURS);
redisTemplate.opsForValue().set("config:" + key, value, 1, TimeUnit.DAYS);
```

## 4. 怎么选方案？

| 场景        | 推荐方案       | 原因           |
| --------- | ---------- | ------------ |
| 商品信息、用户资料 | 延迟双删       | 短暂不一致可接受，性能好 |
| 账户余额、库存   | 分布式锁       | 强一致性要求       |
| 不想改业务代码   | Canal + MQ | 解耦，自动同步      |
